10.01.2007 Kurz vor Ladenschluss (31.12.2006) kann die Version 1.0.0 von betoffice bekannt gegeben werden. Die große Umstellung auf Java 5, Maven 2, Springframework 2 und Hibernate 3 ist vollzogen. Eine auslieferungsfähige Version ist noch nicht entstanden. Das Aufsetzen der Datenbank und das Generieren der Homepage verlangt noch zuviel Handarbeit. Das gluehloch-homepage Projekt hat ebenfalls einen Wandel vollzogen: Es kann als eigenständige Java Applikation verwendet werden und hat die Maven-Plugin Hülse verlassen. 27.04.2007 Das neue betoffice-1.0.1 Release unterstützt die Verwendung einer Derby Datenbank. Die zusätzliche Installation einer Datenbank ist nun nicht mehr notwendig und vereinfacht die Installation der Anwendung erheblich. Zusätzlich spielt die Anwendung das Datenbankschema ein, wenn noch nicht vorhanden. Die TODO Liste für das nächste Release ist gefällt. Die Startup Zeit soll verkürzt werden. Hierzu wir das Hibernate Mapping angepasst und eventuell verschieben sich Funktionen aus den StorageObjects in die Services. Die Swinger Utils nehmen langsam Kontur an. Geplant ist noch eine Ausweitung der Action-Verwaltung (Menü/Kontextmenü-Generierung eventuell mit einem Rechtesystem oder Sichtbarkeitsverwaltung). Gluehloch-homepage wird mit der Javascript Bibliothek Prototype ausgeliefert. Das Tippformular wird auf Prototype umgestellt. Die serverseitige Auswertung ist von PHP4 auf PHP5 umgestellt. Auf register_globals kann dann bald verzichtet werden. Insgesamt sind viele Baustellen geschlossen und eröffnet worden, trotz der wenigen verfügbaren Zeit.
